Actividad Git y GitHub

Git

Git es un sistema de control de versiones distribuido ampliamente utilizado para el seguimiento de cambios en proyectos de software y la colaboración entre desarrolladores.

Aspectos Relevantes

  • Control de versiones

    Git permite a los desarrolladores realizar un seguimiento de los cambios en el código fuente a lo largo del tiempo. Esto es útil para mantener un historial de modificaciones, identificar quién hizo qué cambio y revertir a versiones anteriores si es necesario.

  • Distribuido

    A diferencia de los sistemas de control de versiones centralizados, como Subversion, Git es distribuido. Esto significa que cada clon de un repositorio Git es una copia completa del historial y puede funcionar de manera independiente, lo que lo hace adecuado para el trabajo en equipo y la colaboración en diferentes ubicaciones.

  • Repositorio

    Un repositorio Git es un almacén de todos los archivos, carpetas y su historial de cambios. Puede existir en una plataforma remota, como GitHub, GitLab o Bitbucket, o localmente en la máquina de un desarrollador.

  • Repositorios remotos

    Plataformas como GitHub, GitLab y Bitbucket proporcionan alojamiento en la nube para repositorios Git, lo que facilita la colaboración y el seguimiento de cambios en equipos distribuidos.

GitHub

GitHub es una plataforma de alojamiento y colaboración para proyectos de desarrollo de software basados en el sistema de control de versiones Git. Proporciona herramientas para gestionar y colaborar en proyectos de código abierto y privados.

Aspectos Relevantes

  • Repositorios públicos y privados

    En GitHub, puedes crear repositorios públicos (accesibles para todos) o privados (accesibles solo para las personas que invites). Los repositorios públicos son ideales para proyectos de código abierto, mientras que los privados son útiles para proyectos comerciales o sensibles.

  • Colaboración

    GitHub permite a los equipos trabajar juntos en proyectos. Varios desarrolladores pueden clonar, realizar cambios, crear ramas, fusionar cambios y colaborar en la revisión de código.

  • Wiki y Páginas de GitHub

    Puedes crear wikis para documentar tu proyecto y crear páginas web estáticas a través de la función GitHub Pages, lo que facilita la presentación de documentación, guías de usuario o páginas de inicio de proyectos.

  • Seguridad y revisión de código

    GitHub ofrece características de seguridad, como análisis de dependencias, detección de vulnerabilidades y revisión de código, para garantizar la calidad y seguridad del software.
